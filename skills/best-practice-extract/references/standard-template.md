# 標準ドキュメント 雛形

ステップ5で決定した標準を記録するためのテンプレート。保存先はユーザー指定。
ファイル名は `<theme>-standard.md` を推奨。

```markdown
# 横断標準: <テーマ名>

- **策定日**: YYYY-MM-DD
- **テーマ slug**: <theme>
- **対象リポジトリ**: owner/repo-a, owner/repo-b, ...

## 決定した標準（あるべき姿）

このテーマについて、各リポジトリが従うべき標準を定義する。

- 標準項目1: ...
- 標準項目2: ...
- 標準項目3: ...

（設定ファイル例やスニペットがあればここに置く）

## 根拠（どのリポの何を採用したか）

| 項目 | 採用元 | 理由 |
|---|---|---|
| ... | owner/repo-a | ... の点が優れていた |
| ... | owner/repo-b の統合 | A の簡潔さ + B の網羅性 |

## 適用手順（各リポで何を変えるか）

1. ...
2. ...
3. ...

## 除外・例外条件

- このリポ/条件では標準を適用しない: ...
- 段階移行が必要なケース: ...

## 適用状況

| リポジトリ | 状態 | PR |
|---|---|---|
| owner/repo-a | 適用済 / 未適用 / 対象外 | #123 |
| owner/repo-b | ... | ... |
```
